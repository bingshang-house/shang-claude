# 房地合一稅 2.0 試算規則

> 給未來的我（or 下一個 Claude）：這份是「在屋主面前手機操作」的版本，不要寫得太自動化、不要把屋主一定要選的欄位（自住、調職非自願）藏起來變成 AI 自己決定。**讓賞哥邊問屋主邊勾**才是正確流程。

---

## 0. 第一條鐵律：用 2.0 不要混到 1.0

**110 年 7 月 1 日（含）以後** 簽約成交（移轉登記日）的案子 → **全部走 2.0**。
舊制 1.0 範例 PDF 只用來對照欄位名稱，**稅率/費用率/持有期間級距完全不一樣**，不要混。

| 項目 | 舊制 1.0 | 新制 2.0（**我們用這個**）|
|---|---|---|
| 適用日 | 105/1/1 ~ 110/6/30 | 110/7/1 ~ 至今 |
| 持有 1 年內 | 45% | — |
| 持有 1 ~ 2 年 | 35% | 同 ≤ 2 年 → 45% |
| 持有 2 ~ 5 年 | — | **35%** |
| 持有 5 ~ 10 年 | 20% | **20%** |
| 持有 10 ~ 20 年 | 15% | **15%** |
| 持有 20 年以上 | 15% | **15%**（無級距變化）|
| 未提示證明的可減除費用 | 成交價 × **5%** | 成交價 × **3%**，**上限 30 萬** |
| 自住免稅額 | 400 萬 | 400 萬（不變）|
| 自住稅率 | 10% | 10%（不變）|

⚠️ **2.0 把「未提示費用率」從 5% 砍到 3% 並加 30 萬上限**。屋主自己估算時最常套錯這條。

## 1. 從謄本自動帶入的欄位（house-profile.html 已解析）

這幾欄不用手填，直接從建物/土地謄本所有權部抓：

| 欄位 | 謄本來源 | 申報書對應 |
|---|---|---|
| **所有權人姓名** | 所有權部 → 登記名義人 | 納稅義務人 (原所有權人) → 姓名 |
| **取得日期** | 所有權部 → 登記日期 | 「取得日期」欄（持有期間 RR 起算）|
| **取得原因** | 所有權部 → 登記原因 | 「取得原因」欄（轉成代號 a/b/c/d/e/f/h/z）|
| 房屋稅籍 / 段小段建號 | 建物標示部 | 房屋稅籍編號 / 段小段建號 |
| 土地坐落 | 土地謄本標示部 | 土地坐落（鄉鎮市區/段/小段/地號）|

### 登記原因 → 代號對照（謄本中文 → 申報書代號）

```
買賣           → a
拍賣 / 強制執行 → b
繼承           → c
遺贈           → d
贈與           → e（一般贈與）
夫妻贈與        → f（配偶贈與，需另填配偶原始取得原因）
廢止徵收        → h
其他（信託歸屬、判決移轉、共有物分割、第一次登記…）→ z
```

⚠️ **代號 g 在 2.0 已不用**（舊制是「自地自建」），自地自建在 2.0 改成案件類型勾選。

### 取得日認定表（2.0 版重點，prototype 已實作 hint）

| 登記原因 | 取得日認定 | 持有期間特殊規則 |
|---|---|---|
| **a 買賣** | 移轉登記日 | — |
| **b 拍賣 / 強制執行** | **領得權利移轉證書日**（通常早於登記日 1-2 個月） | — |
| **c 繼承** | **被繼承人死亡日**（不是繼承登記日！） | **可併計被繼承人持有期間**；連續繼承（祖父→父→子）可累計持有期間（2023/11 函釋放寬） |
| **d 受遺贈** | 被遺贈人死亡日 | 同 c 繼承 |
| **e 一般贈與** | 贈與登記日 | 原始成本 = 贈與當時申報現值總額 |
| **f 配偶贈與** | **自原配偶取得日起算**（追溯到配偶當年怎麼買的） | 同上；**這條超容易錯**，遇到 f 一定要追問屋主配偶當年的取得方式與日期 |
| **h 廢止徵收** | 參閱財政部說明八 | — |

### 都更 / 危老（特殊取得日認定）

| 案件 | 取得日認定 | 稅率持有期間 |
|---|---|---|
| 分配地（土地） | 原土地取得日（更新前） | 同左 |
| 分配房（房屋） | **新房屋登記日（更新後）** | 同左 |
| 超額分配 | 超額部分登記日 | 從超額起算 |
| 自地自建 / 合建分屋 | 房屋建築完成登記日 | **改用土地持有期間算稅率** |

### 新舊制分界（取得日 vs 出售日）

```
取得日 < 民國 105/01/01（西元 2016/01/01）
   → 適用舊制：綜所稅 → 財產交易所得稅（不走房地合一）
   → prototype 偵測到自動跳紅警示，建議手動算

取得日 ≥ 105/01/01：房地合一 1.0 / 2.0
   → 出售日 ≥ 110/07/01 → 走 2.0（不論取得日）
```

⚠️ **prototype 已實作**：取得日 < 105/01/01 → 紅字警示「⚠️ 此案適用舊制（財產交易所得稅），本工具僅供參考，請另查綜所稅算法」。

## 2. 屋主面前要問的手填欄位

按申報書順序，從上到下：

### 2.1 案件類型（勾選，多選）

```
□ 一般案件
   □ 房屋、土地（最常用）
   □ 房屋使用權
   □ 預售屋及其坐落基地
   □ 股份或出資額（特殊案）

□ 符合自住房地減免優惠條件 ← 屋主自己決定，要問！
□ 自住房地

□ 重購案件
   □ 先購後售
   □ 先售後購

□ 獨資或合夥組織交易案件（個人通常不用勾）

□ 因調職或非自願因素交易 ← 重要！稅率變 20%
□ 參與都市更新、危老重建 ← 重要！稅率變 20%
□ 自地自建
□ 合建分售
□ 合建分成
□ 合建分屋
□ 信託

□ 申請核發交易損失證明
```

UI 建議：折疊預設只顯示「一般案件 / 自住 / 重購 / 調職非自願 / 都更危老」5 個常見勾選，其他案件類型放「進階」按鈕展開。

### 2.2 納稅義務人居住身分（勾選）

```
□ 居住者（年滿 183 天，本國籍多半是）
□ 非居住者（外國買家 / 海外台商）
□ 國籍：中華民國 / 其他____
```

非居住者 → 直接 35% (>2年) / 45% (≤2年) **無分級**，沒有自住優惠。

### 2.3 金額類（最重要，也最容易算錯）

| 欄位 | 縮寫 | 說明 | 預設帶入 |
|---|---|---|---|
| 成交價額 | **AA** | 這次賣多少 | 從 house-profile 的「成交總價」帶入，可改 |
| 可減除成本 | **ZA** | 附表二合計 | 子表展開填 |
| 可減除移轉費用 | **ZB** | 附表三合計 | 子表展開填 |

成交價如果建物與土地沒有分開，可以**只填一欄合計**（附表一備註）。

## 3. 計算公式（縮寫對照 + 順序）

```
步驟 1：交易所得額
  AA 成交價額 - ZA 可減除成本 - ZB 可減除移轉費用 = GI 交易所得額（或損失）

步驟 2：課稅所得
  GI - ZC 前3年內交易損失 - 土地漲價總數額 = GL 課稅所得
  ※ GL < 0 → 填 0
  ※ 土地漲價總數額：土地稅法第30條第1項，公告土地現值計算（從土地增值稅單抓）

步驟 3a：適用自住優惠
  (GL - DM 自住免稅額 400 萬) × 10% = AF 應納稅額
  ※ GL ≤ 400 萬 → AF = 0
  ※ 應納稅額計算到「元」為止，尾數不滿 1 元無條件捨去

步驟 3b：不適用自住
  GL × 稅率 = AF 應納稅額

步驟 4：本次應自行繳納
  AF - ZD 重購自住房地扣抵稅額 = AH 本次應自行繳納稅額
```

### 縮寫總表（看到 AA/ZA/GI 不要慌）

| 縮寫 | 中文 | 出處 |
|---|---|---|
| AA | 成交價額 | 主表 |
| ZA | 可減除成本 | 附表二合計 |
| ZB | 可減除移轉費用 | 附表三合計 |
| GI | 交易所得額（或損失）| AA-ZA-ZB |
| ZC | 前3年內交易損失 | 附表四合計 |
| GL | 課稅所得 | GI-ZC-土地漲價總數額 |
| DM | 自住免稅額 | 固定 400 萬 |
| AF | 應納稅額 | GL × 稅率 |
| ZD | 重購扣抵稅額 | 附表五合計 |
| AH | 本次應自行繳納稅額 | AF-ZD |
| RR | 持有期間 | 取得日 → 交易日 |

## 5. 自住房地優惠資格（屋主面前要問清楚）

**5 個條件全中才算自住**：

1. 出售房屋、土地辦竣**所有權移轉登記日**起算前 6 年內，個人或其配偶、未成年子女**設有戶籍**
2. 交易前 6 年內，**無出租、供營業或執行業務**使用
3. 個人與其配偶及未成年子女於交易前 6 年內**未曾適用本款規定**（一輩子能用一次喔，這條最容易踩雷）
4. 課稅所得超過 400 萬部分才課 10%
5. 房屋、土地必須**同時**符合以上條件

**屋主問答話術建議**（手機 UI 上做成 5 個 yes/no）：
1. 「您或配偶/未成年子女在這個房子的戶籍滿 6 年了嗎？」
2. 「這 6 年內有沒有出租或拿來做生意？」
3. 「您（含配偶/未成年子女）以前賣房有用過自住優惠嗎？」（**只要用過一次就不能再用**）
4. 「這個房子建物與土地是同一人所有嗎？」（如果建物和土地不同人，自住資格分別判斷）

5 題全 No / 全符合 → 顯示「✅ 適用自住優惠，稅率 10%（超過 400 萬部分）」
任一題不符 → 顯示「❌ 不適用自住，回到一般稅率」並列出原因

## 6. 附表二（ZA 可減除成本）—— 屋主常常少算的部分

```
ZA = ZA1 + ZA2 + ZA3
```

### ZA1：原始取得成本

**出價取得**（買賣、拍賣）：
- 當初買進的成交價 + 必要費用

**繼承、受贈取得**：
- 繼承/受贈日的房屋評定現值 + 公告土地現值
- × 消費者物價指數調整（依交易日的物價總指數）

**必要費用**（取得時的）：
- 契稅
- 印花稅
- 代書費
- 規費
- 仲介費（買進時付的）
- 公證費
- 增置、改良、修繕費（**這條超多人忘記！**裝潢、加蓋、廚房翻新、衛浴改造都算，要有發票）

### ZA2：改良土地費用

地方稅稽徵機關核准減除的改良土地費用（從土地增值稅繳款書「改良土地費用」欄抄）。

### ZA3（罕見）

申報書沒明列細項，留 0 即可。

## 7. 附表三（ZB 可減除移轉費用）—— **2.0 改了，注意 3% / 30 萬**

```
ZB = max(ZB1 + ZB3, ZB2 + ZB3)
```

### ZB1：已提示證明文件

**出售時付的**：
- 仲介費（屋主賣這次付給我們的）
- 代書費（過戶代書費）
- 規費（謄本費、登記規費）

⚠️ **2.0 沒有「清潔費」**了（1.0 有）。

### ZB2：未提示證明文件

```
ZB2 = min(成交價額 × 3%, 300,000)
```

⚠️ **2.0 是 3% 不是 5%！而且封頂 30 萬！**
舊制 1.0 是 5% 沒上限，賞哥要記得，1 億成交價 1.0 算 500 萬未提示費用，2.0 只算 30 萬，差 470 萬。

### ZB3：土地增值稅可減除部分

申報移轉現值**超過**公告土地現值的部分對應的土地增值稅。
> 土地增值稅本來是按公告現值計算，如果申報現值高於公告現值，超出部分繳的土增稅可以列為移轉費用。
> 從土地增值稅繳款書算。

### ZB 取較高者邏輯

實作時兩個都算，取大者：
```js
const zb = Math.max(zb1 + zb3, zb2 + zb3);
```

## 8. 附表四（ZC 前 3 年交易損失）

3 年內賣過其他房地有損失（GI < 0）且**已申報核定**的，可以拿來抵這次。
要有國稅局核定文號（檔案編號）。

實作上做成「+ 新增前次損失」按鈕，每筆填：
- 核定日期
- 檔案編號
- 交易日期
- 核定交易損失金額

通常屋主沒有，預設 0。

## 9. 附表五（ZD 重購自住扣抵）

只有勾「重購案件」才需要填。

```
條件：出售自住房地後 2 年內重購自住房地，或重購自住房地後 2 年內出售原房地

(1) = 重購價額
(2) = 出售價額
AF = 出售自住房地的應納稅額（從步驟 3 算來）

(1) ≥ (2) → ZD = AF（全額退）
(1) < (2) → ZD = AF × (1)/(2)（按比例退）
```

⚠️ 重購扣抵後 5 年內不得改作自住以外之用、不得移轉，否則會被追繳。

## 10. 土地漲價總數額 + 土地增值稅（步驟 2 用 + 完整算法）

### 10.0 兩個重點先記住

1. **房地合一稅裡，只有「土地漲價總數額」可以扣**，不是「土地增值稅本身」。屋主常以為「我繳了 50 萬土增稅可以扣」，**這是錯的**，能扣的是漲價那塊金額（通常更大）。
2. **賣方繳土增稅，買方繳契稅**。這份規則只算賣方端。

### 10.1 漲價總數額公式

```
申報移轉現值（A）= 公告土地現值 × 土地總面積 × 權利範圍
調整後前次現值（B）= 前次移轉現值 × CPI指數 × 土地總面積 × 權利範圍
土地漲價總數額（a）= A − B
   ※ a < 0 → 視為 0（沒漲價沒土增稅）
```

⚠️ **捨入規則（踩過坑）**：

| 計算階段 | 規則 |
|---|---|
| 申報移轉現值 A | **Math.floor**（無條件捨去到整數） |
| 調整後前次現值 B | **Math.floor** |
| 一般稅率最終稅額（a×rate − b×coef） | **Math.round**（四捨五入） |
| 自用 10% 稅額 | **Math.round** |

**Why**：先 floor 各項中間值再相減會差 1 元；正確做法是**全精度連乘到最後一步才 floor / round**。中間用 14.833（3 位小數）會精度損失。

### 10.2 CPI 物價指數從哪來

主計總處每月公布，網址 `https://ws.dgbas.gov.tw/001/Upload/463/relfile/10315/2649/cpispleym.xls`

**注意這是「稅務專用對照表」不是原始 CPI**：基期月對應 100.0，其他月份是換算後比率，**直接乘下去就是「物價指數調整後前次移轉現值」**，不需任何轉換。

本專案已實作自動同步：
- GitHub Actions cron `.github/workflows/cpi-sync.yml` 每月 1 號台北 11:17 抓
- 解析 script `scripts/fetch-cpi.py`（**xlrd==1.2.0**，2.0+ 不支援 .xls）
- 輸出 `data/cpi-tax-index.json`，68 年資料（民國 48-115）
- 也支援 workflow_dispatch 手動觸發

對照已驗證一致：

| 民國年/月 | 指數 | 對照來源 |
|---|---|---|
| 115/03 | 100.0 | 基期月（最新） |
| 111/10 | 106.4 | etax 試算頁面（旗津 0900-* 案 3 筆） |
| 79/05 | 174.4 | etax 試算頁面（楠梓和平段 0233-0001 案） |

### 10.3 一般稅率速算表（土稅 §33，含長期持有減徵）

| 持有年限 | 第 1 級（漲幅 ≤ 1 倍） | 第 2 級（漲幅 1-2 倍） | 第 3 級（漲幅 > 2 倍） |
|---|---|---|---|
| **< 20 年** | a × **20%** | a × **30%** − b × **10%** | a × **40%** − b × **30%** |
| **20-30 年** | a × 20% | a × 28% − b × 8% | a × 36% − b × 24% |
| **30-40 年** | a × 20% | a × 27% − b × 7% | a × 34% − b × 21% |
| **> 40 年** | a × 20% | a × 26% − b × 6% | a × 32% − b × 18% |

a = 漲價總數額；b = 調整後前次現值總額。**第 1 級不減徵**（怎麼持有都是 20%）。

### 10.4 自用住宅 10% 優惠（土稅 §34）

**§34 比房地合一的「自住」更寬鬆**：只要這次戶籍在、最近 1 年沒出租，且**一生一次**沒用過，就能套。

但有面積上限：

| 土地類型 | 上限 |
|---|---|
| 都市土地 | **3 公畝（300 ㎡）** |
| 非都市土地 | **7 公畝（700 ㎡）** |

**超過上限 → 拆分計算**（按面積比例）：
```js
const eligibleRatio = limit / landAreaShare;        // 上限佔比
const eligiblePart  = landGain × eligibleRatio × 0.10;
const excessPart    = generalTax × (1 - eligibleRatio);
selfUseTax = round(eligiblePart + excessPart);
```

「**一生一屋**」上限更嚴：都市 1.5 公畝 / 非都市 3.5 公畝（條件嚴格，預設用「一生一次」）。

### 10.5 都市 / 非都市自動判斷

從謄本 + GIS 推（house-profile.html 已有 GIS 自動帶入）：

| 訊號 | 判定 |
|---|---|
| GIS 有 `nonUrbanZone` / `nonUrbanUse`（一般農業區/特定農業區/山坡地保育區/森林區/鄉村區） | 非都市 |
| 謄本「使用地類別」有值（甲建/乙建/農牧/林業） | 非都市 |
| 兩者都空白 | **預設都市**（同 etax 試算頁面預設） |

### 10.6 免徵 / 不課徵類型偵測（土稅 §39 系列）

從謄本「使用分區」/「使用地類別」推：

| 關鍵字 | 法源 | UI 處理 |
|---|---|---|
| `道路 \| 公園 \| 綠地 \| 廣場 \| 學校 \| 公共設施保留` | 公保地（土稅 §39 II）→ 免徵 | 稅額照算 + 旁註紅字「⚠️ 公保地，賣方有機會申請免徵」 |
| `農業 \| 農地 \| 特定農業 \| 一般農業 \| 山坡地保育 \| 森林 \| 林業` | 農地（土稅 §39-2）→ 不課徵 | 稅額照算 + 旁註紅字「⚠️ 農地，賣方有機會申請不課徵」 |

**最終認定權在稅捐處**，工具只做提示。詳細條文見 `memory/reference_land_tax_laws.md`。

### 10.7 房地合一裡只能扣「漲價總數額」這一塊

> 公式：房地合一課稅所得 = 交易所得 − 前 3 年損失 − **土地漲價總數額**
> 不是 − 土地增值稅！屋主自己估算時最常套錯這條。

實作上：
- house-profile.html 解析土地謄本「前次移轉現值或原規定地價」表格 → 算出漲價總數額 → 寫進房地合一試算
- prototype 端：上傳土地謄本 → 自動帶入「土地漲價總數額」欄位（ZE 之類）

## 11. 持有期間 RR 計算

```
RR = 交易日 - 取得日
```

- 取得日：謄本登記日期（除特殊案 c/d/f/h，另查說明七、八）
- 交易日：所有權移轉登記日（**不是簽約日**！）
- 持有期間以「年」計算，超過 1 日不滿 1 年算進該年

實作時用 dayjs / Date：
```js
const rrYears = (txnDate - acquireDate) / (365.25 * 86400 * 1000);
```

級距判斷：
```js
if (rrYears <= 2) bracket = '≤2年';
else if (rrYears <= 5) bracket = '2-5年';
else if (rrYears <= 10) bracket = '5-10年';
else bracket = '>10年';
```

## 13. 容易踩雷的地方（給未來的 Claude）

1. **2.0 vs 1.0 混用**：寫程式時稅率表硬碼，**只用 2.0**。1.0 案子幾乎絕跡了，要是真碰到舊案，提示屋主「這是舊制，請另外算」。
2. **未提示費用率寫成 5%**：是 **3%，且封頂 30 萬**。
3. **持有期間用簽約日算**：要用**移轉登記日**。
4. **f 配偶贈與取得日填錯**：要追到「配偶第 1 次相互贈與前的原始取得日」。
5. **自住優惠 6 年一輩子一次**：屋主常忘記以前賣房用過。
6. **GL < 0 沒設成 0**：課稅所得不能負，要 clamp 0。
7. **AF 尾數沒捨去**：規定「不滿 1 元無條件捨去」。
8. **重購扣抵的 (1)(2) 算錯**：(1)=重購價額，(2)=出售價額。
9. **建物與土地同人才同時自住**：分屬不同人時資格分判。
10. **非居住者沒有自住優惠也沒有分級**：勾居住身分時要 lock 掉自住勾選。

## 14. 業務預設值（賞哥規則 / prototype 自動帶）

| 欄位 | 預設值 | 觸發條件 |
|---|---|---|
| **仲介費**（出售時 ZB1） | 售價 × **4%** | AA 變動觸發；手動改後鎖定（dirty flag） |
| **履保費**（出售時 ZB1） | 售價 × **萬分之三**（0.0003） | AA 變動觸發；手動改後鎖定 |
| **履保費**（買進時 ZA） | 取得價 × **萬分之三** | **僅取得原因 = a 買賣** 才帶；繼承/贈與/拍賣不帶（屋主沒繳履保） |

**Why 履保費限制**：
- 履約保證費是「過戶服務費」，只有買賣交易才有
- 繼承（c）、贈與（e/f）、拍賣（b）、廢止徵收（h）→ 沒有履保費，prototype 自動清空
- 屋主取得原因切換 → fee_lvyue 欄位連動更新

實作 dirty flag 模式（tax-calc-prototype.html 已實作）：
```js
let _zbZhongjieUserEdited = false;
let _zbLvyueUserEdited = false;
let _zaLvyueUserEdited = false;

// AA listener
aa.addEventListener('input', () => {
  if (!_zbZhongjieUserEdited) zbZhongjie.value = round(aa.value * 0.04);
  if (!_zbLvyueUserEdited)    zbLvyue.value    = round(aa.value * 0.0003);
});

// 用戶手動改 → 鎖定
zbZhongjie.addEventListener('input', () => { _zbZhongjieUserEdited = true; });
```
