## 一、為什麼做這個

現在的 `house-profile.html` 每次都是**打開、解析謄本、印資料表、關掉就沒了**。夥伴想回頭改已經做過的案子要重傳謄本重跑一次。接下來還要用這些資料讓 AI 生評估建議書，所以需要：

1. **雲端存檔**：夥伴填完資料可以存下來，下次打開能叫回來修改
2. **權限隔離**：每個夥伴只看得到自己的案子，彼此隔離
3. **為 AI 建議書鋪路**：存下來的結構化資料就是 AI 的輸入
4. **AI Agent 團隊**：多個 sub-agent 分工採集市場資料（591、實價登錄、銀行估價），組合出完整不動產評估建議書

## 二、整體架構（用白話講）

```
夥伴的 Google 帳號 ──進網站就要登入──> 賞好賞資料表網站
                                          │
                                          ├── 登入時自動串 Drive（不是等要存檔才串）
                                          │
                                          ├── 存檔 ──> 夥伴自己的 Google Drive
                                          │          （彼此看不到）
                                          │
                                          ├── 叫出舊案子 ──> 從 Drive 讀回來，回到表單
                                          │
                                          └── AI 建議書 ──> 多個 sub-agent 分工
                                                          （591 / 實價登錄 / 銀行估價 / 四大估價法）
```

**比喻**：網站是「辦公桌」，Google Drive 是「每個人自己的抽屜」。辦公桌要先刷門禁卡（Google 登入 + 白名單）才能進，進來後抽屜自動打開。賞哥控制誰能進辦公室。

## 五-2、安全性決策補充（2026-04-25）

### 為什麼白名單不能寫死 code

事實：`bingshang-house/shang-claude` repo 是 **public**。寫死代表三個風險：
1. git history 公開，任何人 clone 拿到全部歷史名單
2. GitHub Pages 渲染後 F12 直接看到名單
3. 加人要 commit + push，操作面留痕

雖然 Gmail 不是密碼級機密，但「**這群 Gmail 屬於賞好賞團隊**」這個關聯是社交工程的好材料（釣魚信、假冒助理、商業情報）。

### 為什麼選 Worker 不是 Hash 化

考慮過三層保護：

| 層次 | 工時 | 安全度 | 結論 |
|---|---|---|---|
| 1. Hash 化（寫死 code 但 SHA-256 亂碼） | 0 hr | 90% | 攻擊者已知某 Gmail 仍可反向驗證在不在名單 |
| 2. Private repo + GitHub Pages Pro | 0 hr + $4/月 | 95% | 需付費；網頁渲染後仍可 F12 |
| **3. Worker 後端化** ⭐ | +2 hr | 99% | 名單完全離開前端、Dashboard 即改即生效 |

選層次 3 的 +2 hr 換到「未來幾年的維護便利度」+「F12 完全看不到」，划算。

### 為什麼通知信用 EmailJS 不用 NAS 自架

賞哥提議用 QNAP NAS 自架 mail server。技術可行但**家用網路寄信五關陷阱**：
1. **Port 25 被封**（中華電信全國政策防垃圾信，外送 25 出不去）
2. **動態 IP** — Gmail 看到「會變的 IP」直接退信
3. **反向 DNS** — `xxx.dynamic.hinet.net` 一看就家用 → 進垃圾信
4. **SPF / DKIM / DMARC 三大認證** 要全部設好，少一道進垃圾
5. **NAS 斷電 / 斷網** → 申請信寄不出，賞哥不知道

加上「CF Worker → 家用 NAS」需要 NAS 對外開 port，QNAP 近年勒索病毒（DeadBolt、Qlocker）猖獗，**少對外開 port 是黃金法則**。

EmailJS 對「月寄 5 封通知信」是完美對位：5 分鐘設定、永遠免費（200 封/月）、不開 NAS port。

未來若需大量寄信（電子報、行銷）再評估自架；通知信這個量級用 SaaS 是正解。

### 為什麼 Cloudflare Workers 免費

CF 主業是賣大企業 CDN + DDoS 防護（月費 $200~$20 萬）。Workers 是「附餐」拉開發者進生態系；CF 全球 300+ 邊緣節點本來就在處理 CDN 流量，Workers 順便跑邊際成本接近零。免費版每天 10 萬次請求，賞哥的白名單檢查每月用量約 30-100 次，**用到 2050 年都用不完**。

### 三項額外資安衛生（不管選哪方案都該做）

1. ✅ **GitHub 帳號開 2FA**（跟本專案無關都該做）
2. ✅ **謄本 PDF 永遠不存 Drive**（藍圖既定）
3. ✅ **Drive 權限只用 `drive.file` scope**（藍圖既定，只能動自己建的檔）

| 白名單 | 准許使用者名單（像大樓訪客登記簿） |
| API | 網站跟另一個系統溝通的橋樑 |
| JSON | 結構化的資料格式，人類讀得懂的文字（不是 PDF、不是 Word） |
| Google Drive `drive.file` 權限 | 本網站**只能**讀／寫它自己建的檔案，碰不到夥伴的其他 Drive 內容 |
| Sub-Agent | 主 AI 底下的專項 AI 小隊員，各司其職（例：一個抓 591、一個查實登） |
| Orchestrator | 「指揮官」AI，彙總 sub-agent 的結果做最終決策 |
| EmailJS | 純前端寄 email 的服務（不用架後端） |
| Apps Script | Google 官方的 Sheets / Drive 後端腳本語言（免費） |
