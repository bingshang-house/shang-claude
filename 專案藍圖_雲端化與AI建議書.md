## 一、為什麼做這個

現在的 `house-profile.html` 每次都是**打開、解析謄本、印資料表、關掉就沒了**。夥伴想回頭改已經做過的案子要重傳謄本重跑一次。接下來還要用這些資料讓 AI 生評估建議書，所以需要：

1. **雲端存檔**：夥伴填完資料可以存下來，下次打開能叫回來修改
2. **權限隔離**：每個夥伴只看得到自己的案子，彼此隔離
3. **為 AI 建議書鋪路**：存下來的結構化資料就是 AI 的輸入

---

## 二、整體架構（用白話講）

```
夥伴的 Google 帳號 ──登入──> 賞好賞資料表網站
                                    │
                                    ├── 存檔 ──> 夥伴自己的 Google Drive
                                    │          （彼此看不到）
                                    │
                                    └── 叫出舊案子 ──> 從 Drive 讀回來，回到表單
```

**比喻**：網站是「辦公桌」，Google Drive 是「每個人自己的抽屜」。辦公桌是公用的、但抽屜上鎖，鑰匙在夥伴自己手上。賞哥控制誰能進辦公室（白名單）。

---

## 三、路線圖（三階段）

### 🟢 階段 1｜雲端存檔 + 白名單控管

**目標**：夥伴登入後能存案子、叫出來修改；非允許名單內的人進不來。

**交付物（做完能看到什麼）**：
- 網站右上角多「登入 Google」按鈕
- 登入成功後顯示「Hi XXX」+ 兩個按鈕：「☁ 儲存案子」「📂 我的物件清單」
- 儲存後 Google Drive 會多一個叫「賞好賞物件資料」資料夾，裡面每筆案子一個 JSON 檔
- 「我的物件清單」能看到所有存過的案子（依時間或地址列表），點進去就載回表單
- 非白名單 email 登入 → 彈出「請聯絡江經紀人開通」，功能全鎖

**工時估算**：6-8 小時

### 🟡 階段 2｜AI 建議書生成

**目標**：把存好的資料表交給 AI，產出一份「市場分析 + 定價建議 + 賣點文案」的銷售建議書。

**交付物**：
- 資料表下方多「🤖 AI 生成建議書」按鈕
- 點下去等約 30 秒，跳出建議書預覽（HTML 格式）
- 可下載 PDF / Word / 直接列印
- 建議書自動存回 Google Drive 同資料夾（檔名：`1245-0000_建議書.pdf`）

**工時估算**：8-12 小時（含 prompt 調校）

### ⚪ 階段 3｜白名單管理頁（人多再做）

**目標**：人員超過 5 人時，賞哥自己能新增／刪除白名單，不用每次找 Claude。

**交付物**：隱藏管理頁，賞哥專屬登入後能編輯名單。

**工時估算**：2-3 小時（達條件再做）

- [ ] 2. HTML 加 Google 登入按鈕 + 白名單對照邏輯
- [ ] 3. 實作「存檔」：拿資料表 JSON 寫進 Drive

- [ ] 6. 上 GitHub Pages + 線上版驗收

### 驗收標準（SOP）

1. 賞哥用自己 Gmail 登入 → 成功，畫面顯示 Hi
2. 外部 email 登入 → 被擋，顯示「請聯絡江經紀人開通」
3. 存一筆案子 → 去 Google Drive 能看到檔案
4. 關閉分頁、重開網站、重登 → 點「我的物件清單」看到剛存的案子
5. 點清單載回 → 表單內容跟存檔當下一樣
6. 修改後再存 → 同一筆案子更新，不是新增一筆

## 附錄：名詞對照（非 IT 解讀）

| 技術詞 | 白話解釋 |
|---|---|
| OAuth | 「用 Google 帳號登入第三方網站」那個流程（很多網站都用，FB、Line、IG 也是） |
| 白名單 | 准許使用者名單（像大樓訪客登記簿） |
| API | 網站跟另一個系統溝通的橋樑 |
| JSON | 結構化的資料格式，人類讀得懂的文字（不是 PDF、不是 Word） |
| Google Drive `drive.file` 權限 | 本網站**只能**讀／寫它自己建的檔案，碰不到夥伴的其他 Drive 內容 |
