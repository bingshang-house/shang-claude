## 一、為什麼做這個

現在的 `house-profile.html` 每次都是**打開、解析謄本、印資料表、關掉就沒了**。夥伴想回頭改已經做過的案子要重傳謄本重跑一次。接下來還要用這些資料讓 AI 生評估建議書，所以需要：

1. **雲端存檔**：夥伴填完資料可以存下來，下次打開能叫回來修改
2. **權限隔離**：每個夥伴只看得到自己的案子，彼此隔離
3. **為 AI 建議書鋪路**：存下來的結構化資料就是 AI 的輸入
4. **AI Agent 團隊**：多個 sub-agent 分工採集市場資料（591、實價登錄、銀行估價），組合出完整不動產評估建議書

## 二、整體架構（用白話講）

```
夥伴的 Google 帳號 ──進網站就要登入──> 賞好賞資料表網站
                                          │
                                          ├── 登入時自動串 Drive（不是等要存檔才串）
                                          │
                                          ├── 存檔 ──> 夥伴自己的 Google Drive
                                          │          （彼此看不到）
                                          │
                                          ├── 叫出舊案子 ──> 從 Drive 讀回來，回到表單
                                          │
                                          └── AI 建議書 ──> 多個 sub-agent 分工
                                                          （591 / 實價登錄 / 銀行估價 / 四大估價法）
```

**比喻**：網站是「辦公桌」，Google Drive 是「每個人自己的抽屜」。辦公桌要先刷門禁卡（Google 登入 + 白名單）才能進，進來後抽屜自動打開。賞哥控制誰能進辦公室。

## 三、路線圖（三階段）

### 🟢 階段 1｜強制登入 + 雲端存檔 + 自助申請白名單

**目標**：進網站就要登入（Gate at Entry）；登入同時串 Drive；非白名單可自助申請。

**交付物（做完能看到什麼）**：
- 打開網址就顯示「請先登入 Google」登入頁（沒登入看不到工具）
- 登入成功 + 在白名單 → 直接進主畫面，右上顯示「👤 XXX [登出]」，Drive 已串好
- 登入成功 + **不在白名單** → 顯示自助申請單（Gmail 自動帶入，填用途送出）→ 系統寄 email 通知賞哥
- 主畫面頂部加「📂 我的案件 ▼」下拉選舊案，點一下載回表單
- 存檔自動寫 Google Drive 的「賞好賞銷售資料」資料夾
- 每筆案子一個 JSON，檔名類似 `瑞祥龍邸_AA1053498.json`

**工時估算**：8-10 小時（比原本 6-8 多，因為加了自助申請流程）

### 🟡 階段 2｜AI 建議書生成（含 Agent 團隊）

**目標**：一鍵生成完整不動產銷售評估建議書，背後由多個 sub-agent 分工採集資料。

詳見「**六、AI Agent 團隊構想（2026-04-25 新增）**」。

**交付物**：
- A4 預覽下方「🤖 AI 生成建議書」按鈕
- 點下去觸發 sub-agent 團隊（一個主 orchestrator + N 個專項 agent）
- 建議書 render 成美觀 HTML，線上看、可編輯、可列印成 PDF
- **不做 PPT**（HTML 可列印 PDF 已能達成可列印需求）
- JSON 主檔存 Drive，編輯後覆寫同檔

**工時估算**：階段 2 因為 Agent 團隊，預估 20-30 小時（比原本 8-12 大幅增加）

### ⚪ 階段 3｜白名單管理頁（人多再做）

**目標**：人員超過 5 人時，賞哥自己能新增／刪除白名單，不用每次找 Claude。

**交付物**：隱藏管理頁，賞哥專屬登入後能編輯名單。

**工時估算**：2-3 小時（達條件再做）

### 決策 4：建議書格式（✅ 已定）

**HTML 為主 + PDF 列印，不做 PPT**

- **主檔**：結構化 JSON 存 Drive（案件資料 + AI 文案 + 圖像 prompt + 插圖 URL）
- **輸出 1**：線上檢視頁（分享網址給客戶）
- **輸出 2**：可編輯版（網頁直接改，改完存回同一 JSON）
- **輸出 3**：純文字 prompt 塊（一鍵複製貼到 DALL-E 出圖，圖下載後回貼建議書）
- **不做 PPT**：
  - 產 PPT 複雜
  - 編輯要 PowerPoint
  - HTML → 列印 PDF 9 成情境夠用
  - 未來若真需要再加 pptxgenjs 出 PPT（加新 export 函式即可）

## 六、AI Agent 團隊構想（2026-04-25 新增，待詳細討論）

賞哥規劃：**用 Claude Sub-Agent 團隊分工做不動產評估**，不是單一 AI 一次性生成。

### 初步分工構想

| Agent 角色 | 任務 | 輸出 |
|---|---|---|
| **Agent 1：591 比較蒐集** | 抓 591 上當前在售同類型近案件（區 / 坪數 / 格局 / 屋齡） | 清單：地址、總價、單價、開價、屋況、連結 |
| **Agent 2：實價登錄採集** | 查實價登錄一年內同類型、鄰近地區成交 | 清單：地址、成交價、成交日、坪數、車位、建材 |
| **Agent 3：銀行 AI 估價** | 呼叫銀行 AI 智慧估價網站（玉山？華南？）+ **截圖** | 圖片（存進建議書）+ 估價區間 |
| **Agent 4：估價師計算** | 用四大估價法推估合理價位：<br>- 成本法<br>- 收益法<br>- 比較法<br>- 土地重建成本法 | 四個價位 + 推薦最終價 + 每個方法的計算過程 |
| **Agent 主管 / Orchestrator** | 彙總四個 sub-agent 結果，生成最終建議書 | 完整 HTML 建議書 |

### 還要討論的（明天）

- 591 抓取：爬蟲合法性 / API 可用性 / 反爬處理
- 實價登錄：內政部 API 端點 / 資料頻率 / 格式
- 銀行 AI 估價：哪些銀行有公開入口？要登入嗎？截圖用 Playwright？
- 四大估價法：成本法、收益法、比較法、土地重建成本法的公式參數要哪些？
- Sub-agent 協調：怎麼分工、怎麼合併、誰決定最終建議價
- 執行時間：生一份建議書要跑多久？同步等 vs 非同步通知？
- 失敗處理：某個 agent 失敗時整體降級？
- 成本：每份建議書的 Gemini token / 銀行截圖 / 爬蟲成本

## 附錄 A：名詞對照（非 IT 解讀）

| 技術詞 | 白話解釋 |
|---|---|
| OAuth | 「用 Google 帳號登入第三方網站」那個流程（FB、Line、IG 也是） |
| 白名單 | 准許使用者名單（像大樓訪客登記簿） |
| API | 網站跟另一個系統溝通的橋樑 |
| JSON | 結構化的資料格式，人類讀得懂的文字（不是 PDF、不是 Word） |
| Google Drive `drive.file` 權限 | 本網站**只能**讀／寫它自己建的檔案，碰不到夥伴的其他 Drive 內容 |
| Sub-Agent | 主 AI 底下的專項 AI 小隊員，各司其職（例：一個抓 591、一個查實登） |
| Orchestrator | 「指揮官」AI，彙總 sub-agent 的結果做最終決策 |
| EmailJS | 純前端寄 email 的服務（不用架後端） |
| Apps Script | Google 官方的 Sheets / Drive 後端腳本語言（免費） |

## 附錄 B：技術決策記錄（已確認，不會再動）
