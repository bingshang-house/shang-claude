1API 金鑰防盜用

金鑰的保護是可以做到很紮實的，這是重點投資方向。

1-1 不支援網域鎖的 key → 一律搬後端

Key會不會外洩對策
Google Maps 有綁 github.io 網域，偷到也打不了 ✅ 現狀 OK
Gemini 不支援網域鎖，偷到就能用 🔧 搬 Cloudflare Worker
TGOS 申請到後看支不支援鎖 若不支援就也走 Worker
原則：產一把新 key 前，先問一句 「這個 API 支不支援綁網域/IP？不支援就放後端」。

1-2 Google Cloud 設「每日花費上限」

即使 key 被盜，也不會一次刷你幾萬塊。這是最後一道保險。

  1. Google Cloud Console → Billing
  2. → Budgets & Alerts
  3. → 設每日 US$5 上限
  4. 到上限自動停所有 API，隔天恢復

1-3 最小權限：key 只開必要的 API

Google Cloud Console → API 憑證 → 每把 key 點進去 → 只勾這把 key 會用到的 API，其他全關。

  • Maps key → 只勾 Maps JavaScript API + Places API (New)
  • Gemini key → 只勾 Generative Language API

1-4 Cloudflare Worker 加「來源檢查」

現在你的 Worker 網址 kcg-urban-proxy.bingshang1019.workers.dev 是公開的，任何人都能打。

加一行 Origin 檢查，就能擋掉 99% 的亂用：

const origin = request.headers.get('Origin') || '';
const allowed = ['https://bingshang-house.github.io', 'http://localhost'];
if (!allowed.some(o => origin.startsWith(o))) {
  return new Response('Forbidden', { status: 403 });
}
⚠️ 注意：Origin 是瀏覽器加的，用 curl 可以偽造 → 所以還要搭 Cloudflare Rate Limiting（免費方案每 10 秒限制幾次），雙保險。

1-5 定期檢查用量

每週花 30 秒到：

  • Google Cloud Console → Metrics：看 Gemini / Maps 呼叫次數有沒有異常飆高
  • Cloudflare Workers → Analytics：看 Worker 呼叫量
不正常就立刻「revoke 舊 key + 產新 key」。
走 Worker 的好處：前端不用改，只要改 Worker 環境變數就好。

2網頁設計防盜用

誠實說：前端的本質是「瀏覽器必須下載你的 HTML/CSS/JS 才能跑」， 任何人在瀏覽器按 F12 就看得到，這是技術極限，沒有工具能真正防拷貝。

但可以提高抄走的成本。

2-1 最重要：把「真正值錢的 know-how」搬後端

你這個工具的價值不是 HTML 版面，而是：

  • 📄 PDF 解析 regex 的血淚經驗
  • 🗺️ GIS 圖層 16 / 22 / 13 怎麼組合
  • 🏗️ buildmis 的 dkey → JSON 端點鏈
  • 🎓 學區資料庫
這些東西只要都在 Worker 裡，即使別人抄走 HTML 也跑不起來（因為沒有 Worker）。 目前 PDF 解析還在前端，將來可以慢慢搬。

2-3 JS 壓縮 + 混淆（可選）

用 Webpack 或 esbuild 把 JS 壓縮混淆後才部署，抄走的人看到是：

function _0x3a1b(a,b){var c=a+b;return c^0x1234;}

逆向回有意義的程式碼要花好幾個小時。

代價：自己 debug 也變痛苦，除非真的有人在抄，否則不急。

2-4 網站加 noindex 或改登入制

如果你覺得這工具是內部用的，不希望被 Google 搜到：

  • HTML 頂部加 <meta name="robots" content="noindex, nofollow"> → Google 不收錄
  • 或套一層「輸入密碼才能進」（Cloudflare Access 免費方案可做）

2-5 DMCA 投訴（發現抄襲時的武器）

真的發現有人整份照抄：

  • 對方放 GitHub → 向 GitHub 提 DMCA takedown（1-3 天會下架）
  • 對方放自己網站 → 向他的主機商（Cloudflare、AWS）投訴
  • 證據就是 2-2 那段版權註解

4金鑰被洩漏時的緊急處理 SOP

症狀：收到 Google 來信說「your API key was leaked」，或呼叫 API 收到 403 PERMISSION_DENIED。
  1. 立刻到 Google Cloud Console 把該 key 刪掉（即使已被 Google 自動停用，也要自己刪乾淨）
  2. 產新 key
  3. 檢查新 key 的網域/API 限制是否跟舊的一樣
  4. 更新 Worker 的 Secret（不要改 HTML）
  5. Cloudflare Worker 重新 Deploy
  6. 測試功能恢復正常
  7. 檢查 Billing 有沒有異常扣款 → 有的話立刻開 Google Cloud Support 申訴
